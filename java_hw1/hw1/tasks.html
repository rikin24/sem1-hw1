Homework 1: Tasks for rb549

Your personal task is to develop a system to manage modules and students taking these modules.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Java Classes [13 marks]

Create a Java class named co2103.hw1.domain.Student with the following private fields:

  1. String name
  2. String facebook
  3. String program
  4. int hours
Generate necessary getters and setters. [7 marks]

Create a Java class named co2103.hw1.domain.Module with the following private fields:

  1. int id
  2. String title
  3. String convenor
  4. List<Student> students
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists in the project). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Module> modules in class co2103.hw1.Hw1Application. Initialise the list inside class co2103.hw1.Hw1Application to contain one module with two students. [4 marks]

Create controller co2103.hw1.controller.ModuleController and add methods with request mappings for the following behaviour:

  1. GET requests for URL "/modules" should show view modules/list. [2 marks]
    • Add the list of all modules from co2103.hw1.Hw1Application to the model as attribute modules. [2 marks]
    • Create the corresponding JSP to list the title and convenor of each module followed by a link to URL "/students?module={id of module}", e.g., "/students?module=3". [4 marks]
  2. Requests for URL "/newModule" should show view modules/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="module" and action="/addModule" and inputs for all fields of class Module except for students. [5 marks]
  3. POST requests for URL "/addModule" should add a new module to the list of all modules in co2103.hw1.Hw1Application and redirect to /modules. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller class co2103.hw1.controller.StudentController and add methods with request mappings for the following behaviour:

  1. GET requests for URL "/students" should show view students/list. [2 marks]
    • Based on request parameter module determine the corresponding module from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all students of that module to the model as attribute students. [2 marks]
    • Create the corresponding JSP for the view to list the name, facebook, and program of each student. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newStudent?module={id of module whose students are listed}", e.g., "/newStudent?module=3". [4 marks]
  2. Requests for URL "/newStudent" should show view students/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="student" and action="/addStudent" and inputs for all fields of class Student. [6 marks]
    • Also add the request parameter module with the ID of the current module. [4 marks]
  3. POST requests for URL "/addStudent" should add a new student to the list of students of the module identified by request parameter module and then redirect to /modules. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Module and make it available in controller ModuleController. [4 marks]

When adding a new module validate that

  1. The value of field id is unique within the list Hw1Application.modules. [3 marks]
  2. The value of field title is not empty. [2 marks]
  3. The value of field convenor is not empty and contains 10 or more characters. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Student and make it available in controller StudentController. [4 marks]

When adding a new student validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field facebook is not empty. [2 marks]
  3. The value of field program is either "Computer Science", "Creative Computing", or "Software Engineering". [3 marks]
  4. The value of field hours is between (including) 0 and 30. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
